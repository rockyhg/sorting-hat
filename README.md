# 組分けアプリ（Sorting Hat）

複数メンバーをグループに分ける組分けアプリ。
過去の組分け履歴を考慮し、同じメンバー同士ができるだけ重複しないようにグループを自動生成します。

## 特徴
- **履歴ベースの重複回避**: 過去の組み合わせを学習し、重複を最小化
- **柔軟なグループ設定**: 人数指定またはグループ数指定を選択可能
- **シンプルな操作性**: テキストエリアにメンバーを入力し、ボタン一つで実行
- **ページ遷移なし**: htmx による非同期通信でスムーズな操作感

## 技術スタック
- **バックエンド**: FastAPI
- **テンプレート**: Jinja2
- **フロントエンド**: htmx + Simple.css
- **データ保存**: JSONファイル (`data/history.json`)

## 実行方法

### 1. 依存パッケージのインストール
```bash
uv sync
```

### 2. アプリの起動
```bash
uv run uvicorn app:app --reload
```

### 3. ブラウザでアクセス
```
http://localhost:8000
```

## 使い方

1. **メンバー入力**: テキストエリアに1行1名でメンバー名を入力
2. **グループ設定**:
   - **人数指定**: 各グループの人数を入力
   - **グループ数指定**: 分けたいグループの数を入力
3. **実行**: 「組分けを実行」ボタンをクリック
4. **結果確認**: 結果がページ遷移なしでその場に表示されます
5. **履歴**: 過去の組分け結果は自動保存され、いつでも確認・削除可能（一括削除も可能）

## データ構造

組分け履歴は `data/history.json` に保存されます:
```json
{
  "history": [
    {
      "id": "uuid",
      "date": "2026-03-20T10:00:00",
      "members": ["Alice", "Bob", "Carol", ...],
      "groups": [["Alice", "Bob"], ["Carol", "Dave"]],
      "group_size": 2,
      "num_groups": null
    }
  ]
}
```

## 組分けアルゴリズム

1. **ペア行列構築**: 履歴から各ペア(A, B)が同じグループになった回数をカウント
2. **初期割り当て**: メンバーをシャッフルしてグループにランダム配置
3. **局所探索最適化**:
   - 異なるグループ間で2人をスワップ
   - スワップによりスコア（全ペアの重複回数合計）が改善するなら採用
   - 改善がなくなるまで繰り返す
4. **複数回試行**: ランダム初期配置で数回試行し、最良の結果を採用
